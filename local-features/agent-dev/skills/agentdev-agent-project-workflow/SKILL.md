# Agent Project Workflow

这个技能服务于 `项目开发` 形态。

## 你要优化的不是“代码量”，而是“进入项目开发的边界是否清楚”

用户进入项目态时，应当明确感受到：

- 这是从装配态升级来的工程化入口
- 会初始化项目目录
- 会保留需求、约束、计划挂载能力等项目级信息
- 后续可以再接更精细的工作流编排

## 当前阶段的目标

- 整理 Agent 名称、目标能力、目标用户、运行形态、限制条件
- 初始化项目目录
- 进入可继续开发的会话
- 不在这个阶段实现复杂、框架无关的工作流编排

## 你的工作方式

- 先确认项目草稿是否清楚
- 再决定是否初始化目录
- 初始化之后，优先让用户进入持续开发会话
- 如果有能力缺口，写 handoff，不要把 Feature 开发强塞到当前路径里

## 当前项目态的边界

这里可以做：

- 项目初始化
- 提示词与 skills 包装
- Agent 基本结构落位
- 项目文档集整理

这里先不做：

- 重型工作流编排
- 与具体业务强耦合的复杂流程系统
- 大量跨框架的 orchestration 细节

## 输出原则

- 先把项目边界和初始化结果说清楚
- 再说明后续从哪里继续开发
- 尽量复用现有 AgentDev / Claw 机制，不额外发明一套体系
